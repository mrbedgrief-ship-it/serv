# CripCraft Local Prototype

## What this is
CripCraft is a **real local prototype** for a Minecraft Java Edition project with a dark-neon multiplayer style. This repository is built for a **complete beginner on Windows**. The goal is simple: get a branded local server running today with a safe hub, a working launcher, and one playable mode.

## Architecture choice
This prototype uses **one single Minecraft server** instead of a proxy network.

Why this choice:
1. It has the fewest failure points for a beginner.
2. It is easier to launch on Windows.
3. It avoids complicated proxy, port, and plugin sync problems.
4. It still allows a real hub + one playable mode inside the same server.
5. It makes the local login fix simple and honest: `online-mode=false` for localhost testing only.

## Local login fix
This project is now configured for **local offline testing**.
That means:
- `server/template/server.properties` uses `online-mode=false`.
- The startup script rewrites `server/runtime/server.properties` to `online-mode=false` every time before launch.
- There is **no proxy**, **no auth plugin**, and **no forwarding config** in this prototype.
- This change is only for testing on your own PC or local network, not for a public internet server.

## What is already working
- A Windows-first launcher UI powered by PowerShell.
- A setup script that downloads the official Minecraft server jar.
- A local `localhost` server startup script.
- A CripCraft hub generated by a datapack.
- Safe spawn and beginner guidance messages.
- A real playable mode called **Zone Survival**.
- A return-to-hub button inside the mode area.
- Rank scaffolding for these planned ranks:
  - Player
  - Crip
  - Neon
  - Legend
  - Overlord
- Local testing mode that disables username verification so you can join without the "Failed to verify username" error.

## Honest audit
This prototype is intentionally small.
- There is **one server only**.
- There is **no BungeeCord**, **Waterfall**, or **Velocity** proxy here.
- There is **no Paper**, **Spigot**, `spigot.yml`, `paper-global.yml`, or auth plugin included.
- Because those files and tools are not present, there was nothing else to reconfigure for forwarding or premium-only login.
- The real root cause was simply that the vanilla template had `online-mode=true`, which blocks local offline testing.

## What is not finished
- No packaged `.exe` launcher yet.
- No online payments or real store integration.
- No plugin-based permissions manager yet.
- No extra maps or advanced quest systems yet.
- No custom resource pack or fancy menu plugin yet.

- `launcher/` - launcher app files.
- `scripts/` - setup, checks, and start scripts.

- `downloads/` - where the official server jar is downloaded.

## First-time setup on Windows
1. Install Java 21.
2. Double-click `scripts/setup.bat`.
3. Wait until the file `downloads/vanilla-server-1.20.4.jar` exists.
4. Double-click `launcher/start-launcher.bat`.
5. Click **Check Files**.
6. Confirm it says local login mode is OK.
7. Click **Play**.
8. If asked about the Minecraft EULA, type `YES` in the server window and press Enter.
9. Wait for the word `Done` in the server window.
10. Open Minecraft Java Edition 1.20.4.
11. Join `localhost`.
12. If `localhost` fails, try `127.0.0.1`.

## How to start the launcher
Double-click:
- `launcher/start-launcher.bat`

## How to start the server manually if needed
Double-click:
- `scripts/start-cripcraft.bat`

## How to join localhost
Inside Minecraft Java Edition:
1. Click **Multiplayer**.
2. Click **Add Server**.
3. Name it anything you want.
4. Server Address: `localhost`
5. If that still fails, change the address to `127.0.0.1`.
6. Join.

## In-game flow
1. You should spawn at the CripCraft hub platform.
2. Read the welcome signs and chat messages.
3. Press the hub button labeled for **Zone Survival**.
4. You will be teleported into the playable mode.
5. Press the return button in the zone bunker to go back to the hub.

## Downloaded file details
This project does **not** include Mojang's server jar inside the repository.
Instead, `scripts/setup.bat` downloads it from Mojang's official version manifest.

Expected filename:
- `downloads/vanilla-server-1.20.4.jar`

## Common errors in plain language

### "Java was not found"
Java is not installed, or Windows cannot find it.
- Install Java 21.
- Close and reopen the launcher.

### "Server jar missing"
The server file was not downloaded yet.
- Run `scripts/setup.bat`.
- Confirm that `downloads/vanilla-server-1.20.4.jar` exists.

### "Failed to verify username" / "Не удалось проверить имя пользователя"
This project is supposed to avoid that error during local testing.
Check these things:
1. Start the server with `launcher/start-launcher.bat` -> **Play**, or `scripts/start-cripcraft.bat`.
2. Let the script create or update `server/runtime/server.properties`.
3. Open `server/runtime/server.properties` and confirm it says `online-mode=false`.
4. Join `localhost`. If that fails, try `127.0.0.1`.
5. Make sure you are joining **this local server**, not an old server folder from somewhere else.

### "The server window closes immediately"
Usually one of these happened:
- Java is missing.
- The EULA was not accepted.
- The server jar is missing.
- Another program is blocking the launch.

Try `Check Files` in the launcher first.

### "Cannot connect to localhost"
Possible reasons:
- The server is still starting.
- The server window was closed.
- You used the wrong Minecraft version.
- Windows Firewall blocked Java the first time.
- Another server is already using port 25565.

### "Spawn looks wrong"
If the world was generated before the datapack finished properly, stop the server and delete the folder:
- `server/runtime/cripcraft_world`

Then start again.
